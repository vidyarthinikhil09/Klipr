## рҹҺҜ Overview

AutoShorts is a production-grade, multi-user platform that automates the entire workflow of creating YouTube Shorts from Twitch clips:

1. **Discover** trending clips from popular streamers via Twitch API
2. **Transcribe** audio using faster-whisper (local, free)
3. **Generate** viral titles, descriptions, and tags using free AI models (OpenRouter)
4. **Render** professional videos with animated word-by-word captions
5. **Upload** directly to YouTube with one click
6. **Learn** from analytics to improve future content

### Why AutoShorts?

| Challenge | AutoShorts Solution |
|-----------|---------------------|
| Finding viral content | Auto-discovers trending clips by view count |
| Writing engaging titles | AI generates click-worthy metadata |
| Professional editing | Animated captions in multiple styles |
| Consistent posting | GitHub Actions for daily automation |
| Improving over time | Analytics feedback loop for self-optimization |

## рҹҡҖ Quick Start

### Prerequisites

- **Python 3.11+** ([Download](https://python.org/downloads))
- **Node.js 18+** ([Download](https://nodejs.org))
- **Git** ([Download](https://git-scm.com))
- **FFmpeg** (for video processing)

### Step 1: Clone Repository

```bash
git clone https://github.com/YOUR_USERNAME/autoshorts.git
cd autoshorts
```

### Step 2: Set Up Supabase

1. Create a free account at [supabase.com](https://supabase.com)
2. Create a new project
3. Go to **Authentication вҶ’ Providers вҶ’ Google** and enable it
4. Go to **SQL Editor** and run the schema from `backend/supabase_schema.sql`
5. Copy your **Project URL** and **anon key** from **Settings вҶ’ API**

### Step 3: Set Up Backend

```bash
# Navigate to backend
cd backend

# Create virtual environment
python -m venv venv

# Activate (Windows)
venv\Scripts\activate

# Activate (macOS/Linux)
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Install pycaps for animated captions
pip install "git+https://github.com/francozanardi/pycaps.git#egg=pycaps[all]"
playwright install chromium

# Copy environment template
cp .env.example .env

# Edit .env with your API keys (see Configuration section)
```

### Step 4: Set Up Frontend

```bash
# Navigate to frontend (from project root)
cd frontend

# Install dependencies
npm install

# Copy environment template
cp .env.example .env

# Edit .env with your Supabase credentials
```

### Step 5: Start Development Servers

**Terminal 1 - Backend:**
```bash
cd backend
venv\Scripts\activate
uvicorn app.main:app --reload --port 8000
```

**Terminal 2 - Frontend:**
```bash
cd frontend
npm run dev
```

### Step 6: Open in Browser

- **Frontend Dashboard:** http://localhost:5173
- **Backend API Docs:** http://localhost:8000/docs

## рҹҺҘ Video Rendering

### Caption Styles

AutoShorts uses [pycaps](https://github.com/francozanardi/pycaps) for professional animated captions:

#### TikTok Style
```css
Font: Montserrat Bold, 52px
Colors: White text, blue highlight (#00D4FF)
Animation: Fade in, scale pulse on highlight
```

#### Hormozi Style
```css
Font: Impact, 58px
Colors: Yellow (#FFFF00) primary, white secondary
Animation: Bold punch-in effect
```

#### MrBeast Style
```css
Font: Bangers, 56px
Colors: Red (#FF0000) / Yellow (#FFFF00) alternating
Animation: Bounce and pop effects
```

#### Ali Abdaal Style
```css
Font: Open Sans, 48px
Colors: White with subtle shadow
Animation: Smooth fade transitions
```

### Rendering Pipeline

1. **Portrait Conversion**: Landscape clips cropped to 9:16 (1080x1920)
2. **Transcription**: faster-whisper extracts word timestamps
3. **Segmentation**: Words grouped (max 3) respecting speech pauses
4. **Caption Generation**: pycaps renders animated subtitles
5. **Export**: MP4 at 30fps, AAC audio

### Performance

| Clip Length | Render Time | Output Size |
|-------------|-------------|-------------|
| 15 seconds | ~8 seconds | ~15 MB |
| 30 seconds | ~15 seconds | ~30 MB |
| 60 seconds | ~30 seconds | ~60 MB |

## рҹ”§ Troubleshooting

### Common Issues

<details>
<summary><b>OpenRouter 404 Error</b></summary>

**Problem:** `No endpoints found for model-name:free`

**Solution:** Free models change frequently. The app auto-retries all available free models. If all fail:
1. Check [openrouter.ai/models](https://openrouter.ai/models) for current free models
2. Update the model list in `backend/app/services/metadata_generator.py`

</details>

<details>
<summary><b>Playwright/asyncio Conflict</b></summary>

**Problem:** `Playwright Sync API inside asyncio loop`

**Solution:** This is handled automatically. pycaps runs in a ThreadPoolExecutor to avoid conflicts with FastAPI's async loop. If you still see this error, ensure you're using the latest renderer.py.

</details>

<details>
<summary><b>Video Not Converting to Portrait</b></summary>

**Problem:** Output video is still landscape

**Solution:** Check that the renderer's `_convert_to_portrait()` method is being called. The conversion happens before pycaps processes the video.

</details>

<details>
<summary><b>Captions Too Large</b></summary>

**Problem:** Text covers too much of the screen

**Solution:** Adjust font sizes in the CSS templates within `renderer.py`. Current sizes are optimized for 1080x1920:
- TikTok: 52px
- Hormozi: 58px
- MrBeast: 56px
- Ali: 48px

</details>

<details>
<summary><b>YouTube Upload Fails</b></summary>

**Problem:** OAuth token expired or invalid

**Solution:**
1. Go to Dashboard вҶ’ YouTube section
2. Click "Connect YouTube"
3. Re-authorize your Google account
4. Tokens are stored per-user in Supabase

</details>

<details>
<summary><b>Twitch Clip Download Fails</b></summary>

**Problem:** yt-dlp can't download clip

**Solution:**
1. Update yt-dlp: `pip install -U yt-dlp`
2. Check if clip URL is valid and public
3. Some clips may be subscriber-only

</details>
